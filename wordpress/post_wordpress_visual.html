Introducción

Cuando desarrollamos una aplicación web, muchas veces necesitamos proteger ciertas partes de nuestra API. Por ejemplo, una zona de usuario, un panel de administración o una ruta que devuelve información privada.

El problema es que una API REST no recuerda por sí sola quién está haciendo cada petición. Cada solicitud es independiente, por lo que necesitamos algún sistema para identificar al usuario después de iniciar sesión.

En este post vamos a resolver ese problema usando JWT. Para que sea más fácil de entender, construiremos una demo visual con Node.js, Express y una pequeña interfaz web con cuatro botones.

¿Qué problema vamos a resolver?

Queremos demostrar cómo una API REST puede tener rutas públicas y rutas protegidas. Una ruta pública puede consultarse sin iniciar sesión, pero una ruta protegida debe comprobar si el usuario tiene un token válido.

El flujo será el siguiente:

  1. El usuario prueba una ruta pública.
  2. El usuario hace login con usuario y contraseña.
  3. El backend genera un token JWT.
  4. El usuario intenta acceder a una ruta protegida sin token y la API lo bloquea.
  5. El usuario accede a la misma ruta enviando el token y la API permite el acceso.

Lo que vas a aprender

  • Crear una API básica con Node.js y Express.
  • Crear una ruta pública.
  • Crear una ruta de login que genera un token JWT.
  • Crear un middleware para proteger rutas.
  • Probar el flujo completo desde una interfaz visual.

Desarrollo paso a paso

1. Crear el proyecto

Primero creamos una carpeta para el proyecto e inicializamos Node.js:

mkdir jwt-api-visual-demo
cd jwt-api-visual-demo
npm init -y

2. Instalar dependencias

Instalamos las dependencias necesarias:

npm install express jsonwebtoken dotenv

También podemos instalar nodemon para trabajar de forma más cómoda:

npm install -D nodemon

3. Crear el servidor

En el archivo index.js configuramos Express, habilitamos JSON y servimos la carpeta pública donde estará la interfaz web.

const express = require('express');
const jwt = require('jsonwebtoken');
require('dotenv').config();

const app = express();
const PORT = process.env.PORT || 3000;
const JWT_SECRET = process.env.JWT_SECRET || 'clave_demo_no_usar_en_produccion';

app.use(express.json());
app.use(express.static('public'));

4. Crear usuario de prueba

Para simplificar la demo, usamos un usuario fijo:

const userDemo = {
  id: 1,
  username: 'daniel',
  password: '1234',
  role: 'user'
};

5. Crear ruta pública

Esta ruta puede consultarse sin autenticación:

app.get('/api/public', (req, res) => {
  res.json({
    ok: true,
    message: 'Esta es una ruta pública. No necesitas token para acceder.'
  });
});

6. Crear login con JWT

La ruta de login recibe usuario y contraseña. Si son correctos, genera un token JWT:

app.post('/api/login', (req, res) => {
  const { username, password } = req.body;

  if (username !== userDemo.username || password !== userDemo.password) {
    return res.status(401).json({
      ok: false,
      message: 'Usuario o contraseña incorrectos'
    });
  }

  const token = jwt.sign(
    {
      id: userDemo.id,
      username: userDemo.username,
      role: userDemo.role
    },
    JWT_SECRET,
    { expiresIn: '1h' }
  );

  res.json({
    ok: true,
    message: 'Login correcto. Token JWT generado.',
    token
  });
});

7. Crear middleware de autenticación

El middleware comprueba si la petición incluye un token válido:

function authenticateToken(req, res, next) {
  const authHeader = req.headers.authorization;
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) {
    return res.status(401).json({
      ok: false,
      message: 'Acceso denegado. No se ha enviado ningún token.'
    });
  }

  jwt.verify(token, JWT_SECRET, (err, user) => {
    if (err) {
      return res.status(403).json({
        ok: false,
        message: 'Token inválido o expirado.'
      });
    }

    req.user = user;
    next();
  });
}

8. Crear ruta protegida

Esta ruta solo funciona si el token es válido:

app.get('/api/profile', authenticateToken, (req, res) => {
  res.json({
    ok: true,
    message: 'Has accedido a una ruta protegida usando un token válido.',
    user: req.user
  });
});

9. Crear interfaz visual

Para que la demo sea más sencilla, la interfaz tiene cuatro botones:

  • Probar ruta pública.
  • Hacer login y generar token.
  • Probar perfil sin token.
  • Probar perfil con token.

De esta forma no hace falta utilizar Postman ni Thunder Client durante el vídeo.

Problemas encontrados

Uno de los problemas principales al trabajar con JWT es enviar mal el token. El formato habitual es Authorization: Bearer TOKEN. Si falta la palabra Bearer o el token está mal copiado, la API no permite el acceso.

También es importante configurar correctamente la clave secreta. Si el token se firma con una clave y se intenta validar con otra diferente, la verificación fallará.

Recomendaciones

Esta demo es sencilla, pero en una aplicación real habría que aplicar más medidas de seguridad. Por ejemplo, las contraseñas no deberían guardarse en texto plano, sino cifradas con una librería como bcrypt.

También es recomendable no subir el archivo .env a GitHub, usar HTTPS en producción y no guardar información sensible dentro del token.

Conclusión

JWT es una solución muy utilizada para proteger APIs REST. El funcionamiento básico consiste en generar un token cuando el usuario hace login y exigir ese token cuando quiere acceder a rutas privadas.

Con esta demo hemos visto el flujo completo de una forma visual: ruta pública, login, token generado, acceso denegado sin token y acceso permitido con token.
